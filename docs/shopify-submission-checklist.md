# Shopify Submission Checklist - Truthful Pre-Submission Status

Last updated: June 18, 2026

Strategy: submit as Free Beta / Free MVP. Billing API is intentionally deferred.

## A. Code Ready - PARTIAL

Completed:

- `npm run build` passes locally.
- `npm test` passes locally.
- OAuth start and callback code exists.
- Shopify HMAC verification exists for OAuth and webhooks.
- Access tokens are encrypted at rest.
- Product sync, order sync, sales velocity, supplier mapping, recommendations, and reorder queue APIs exist.
- GDPR webhook routes exist.
- App Bridge CDN and session token attachment are implemented in the frontend.

Still required:

- Real embedded Shopify Admin install must be tested through production or a real tunnel.
- API session-token enforcement is currently backwards-compatible for local/demo requests; production review should verify embedded requests include session tokens.
- Browser QA must be repeated against production URL.

Cannot submit if:

- Real OAuth install cannot complete on a test store.
- Embedded app requests fail inside Shopify Admin.

## B. Config Ready - FAIL

Completed:

- `shopify.app.toml` contains the intended scopes and webhook paths.
- Required environment variable names are documented.

Still required:

- Fill real Partner Dashboard client ID.
- Set real production HTTPS app URL.
- Set real redirect URL.
- Set real webhook URLs.
- Set production environment variables.
- Generate and store a real 32+ character `SESSION_ENCRYPTION_KEY`.
- Configure production database.
- Apply Prisma migrations in production.

Cannot submit if:

- `client_id` is blank.
- app URL or redirect URLs still point to placeholders.
- Partner Dashboard webhooks are not configured.

## C. Production Ready - FAIL

Completed:

- Production build is generated by `npm run build`.
- Node server can serve `dist` assets locally.

Still required:

- Deploy to a real HTTPS production URL.
- Verify `/`, `/auth`, `/auth/callback`, `/api/*`, and webhook endpoints on production.
- Publish privacy policy and terms at real public URLs.
- Confirm production database persistence.
- Confirm logs do not expose secrets or tokens.

Cannot submit if:

- Production URL is not publicly reachable over HTTPS.
- Privacy and Terms URLs are not publicly reachable.
- OAuth flow has not been tested end-to-end on production or a review tunnel.

## D. Listing Ready - FAIL

Completed:

- Free Beta listing draft exists in `docs/app-listing-final.md`.
- Pricing copy no longer claims paid plans, trial, or automatic billing.
- Support email draft is listed.

Still required:

- App icon, 1200x1200 PNG.
- Screenshots from a real test/demo store.
- Banner image.
- Demo screencast or clear review walkthrough.
- Final public privacy policy URL.
- Final public terms URL.
- Category and app listing fields in Partner Dashboard.

Cannot submit if:

- Listing still references paid plans, free trial, or billing that is not implemented.
- Required assets are missing.
- Public legal URLs are not live.

## E. Review Ready - FAIL

Completed:

- Draft review instructions can explain `read_orders` usage.
- Protected customer data explanation is documented in `docs/protected-customer-data.md`.

Still required:

- Create a review test store with sample products, inventory, orders, and supplier data.
- Provide Shopify review team with install steps and test credentials/access.
- Fill Partner Dashboard Protected Customer Data request for `read_orders`.
- Verify app/uninstalled and GDPR webhooks from real Shopify webhook delivery.

Cannot submit if:

- Review team cannot install and test the app.
- Protected customer data request is not completed for `read_orders`.
- The app charges or advertises paid plans without Billing API.

## Final Status

Overall: PARTIAL.

The codebase is closer to review, but the app should not be submitted until production URL, Partner Dashboard configuration, public legal URLs, review store, protected customer data request, and listing assets are complete.
